# Windows 11 桌面待办提醒小工具设计文档

## 1. 项目定位

本项目是一个运行在 Windows 11 电脑上的本地桌面小工具，主要用于快速记录每日待办事项，并通过快捷键、置顶窗口、定时弹出、事项提醒等方式帮助用户管理当天任务。

应用以“轻量、常驻、快速唤出、本地存储”为核心设计目标。

## 2. 核心功能

### 2.1 全局快捷键显示 / 隐藏

用户启动应用后，可以配置一个全局快捷键。

快捷键触发后，应用在以下两种状态之间切换：

1. 显示在桌面最顶层
2. 隐藏到系统托盘或后台

显示时窗口应满足：

* 置顶显示
* 不抢占复杂布局
* 可快速查看今日待办
* 可直接新增、编辑、完成、废弃事项

隐藏时应满足：

* 主窗口不可见
* 应用仍在后台运行
* 事项提醒和定时弹出功能继续生效
* 系统托盘图标仍可操作

### 2.2 待办事项管理

待办事项以列表形式展示，默认展示当天事项。

每条事项支持：

* 新增
* 查看
* 编辑
* 删除
* 标记完成
* 标记废弃
* 设置提醒时间
* 取消提醒时间

事项状态包括：

| 状态  | 说明                 |
| --- | ------------------ |
| 待处理 | 默认状态，代表事项尚未完成或废弃   |
| 已完成 | 用户主动标记完成           |
| 已废弃 | 用户主动标记废弃，不再继承到后续日期 |

列表展示字段：

* 事项标题
* 事项备注
* 状态
* 所属日期
* 提醒时间
* 是否继承自前一天
* 创建时间
* 更新时间

### 2.3 定时界面弹出

应用支持配置一个或多个定时弹出规则。

到达指定时间后，应用自动弹出主界面，并置于最顶层。

定时弹出可用于：

* 每天固定时间打开待办面板
* 每隔一段时间提醒用户查看待办
* 指定某一天某个时间弹出

弹出时行为：

* 如果主窗口已显示，则将窗口置顶并聚焦
* 如果主窗口隐藏，则显示窗口并置顶
* 如果窗口被最小化，则恢复窗口
* 可以播放提示音或显示系统通知

### 2.4 事项专属时间提醒

每条事项可以设置一个专属提醒时间。

例如：

* 今天 10:30 提醒“提交日报”
* 今天 15:00 提醒“联系客户”
* 今天 18:00 提醒“整理文档”

到达提醒时间后：

* 弹出应用窗口
* 高亮对应事项
* 显示提醒弹窗
* 可选择“知道了”“稍后提醒”“标记完成”“废弃”

提醒时间支持：

* 设置为当天具体时间
* 修改提醒时间
* 取消提醒
* 继承事项时保留原提醒时间的小时和分钟

继承到第二天的提醒规则：

* 如果事项设置的是“每天同一时间提醒”，则继承后在第二天同一时间继续提醒
* 如果事项设置的是“一次性提醒”，则继承后默认不自动提醒，除非用户重新设置
* 推荐默认采用“跟随事项每日提醒”，即未完成事项继承到下一天后，仍按同一时间提醒

### 2.5 按天记录与事项继承

应用以日期为单位记录待办事项。

每日待办规则：

1. 当天新增的事项归属于当天
2. 当天标记完成的事项不会继承到下一天
3. 当天标记废弃的事项不会继承到下一天
4. 当天仍为待处理状态的事项会自动继承到下一天
5. 继承后的事项保留标题、备注、提醒设置和来源记录
6. 历史日期的事项记录仍然保留，便于回看

继承示例：

* 6 月 25 日有事项 A、B、C
* A 已完成
* B 已废弃
* C 仍待处理
* 到 6 月 26 日时，系统自动生成 C 的新一天记录
* 6 月 25 日的 C 仍保留为历史未完成记录
* 6 月 26 日的 C 标记为“继承事项”

## 3. 推荐技术架构

### 3.1 技术栈

| 模块    | 技术                                                    |
| ----- | ----------------------------------------------------- |
| 桌面客户端 | .NET 8 + WPF                                          |
| UI 架构 | MVVM                                                  |
| 本地数据库 | SQLite                                                |
| ORM   | Entity Framework Core 或 Dapper                        |
| 系统托盘  | Windows Forms NotifyIcon 或 WPF NotifyIcon 库           |
| 全局快捷键 | Win32 RegisterHotKey                                  |
| 定时调度  | .NET BackgroundService / DispatcherTimer / Quartz.NET |
| 本地配置  | JSON 配置文件 + SQLite                                    |
| 日志    | Serilog                                               |
| 打包发布  | MSIX 或 单文件 exe                                        |

### 3.2 选择 .NET 8 + WPF 的原因

该工具面向 Windows 11 本地桌面环境，需要稳定支持：

* 全局快捷键
* 系统托盘
* 窗口置顶
* 本地数据库
* 后台常驻
* 开机自启动
* 本地通知
* 轻量打包

WPF 在 Windows 桌面工具类应用中成熟度较高，适合快速开发稳定的小工具。

## 5. 模块设计

## 5.1 窗口管理模块

负责主窗口的显示、隐藏、置顶、聚焦和托盘交互。

核心能力：

* 显示主窗口
* 隐藏主窗口
* 切换显示 / 隐藏
* 设置窗口 TopMost
* 窗口居中或停靠到指定位置
* 从托盘菜单打开设置
* 退出应用

窗口状态：

| 状态        | 说明           |
| --------- | ------------ |
| Hidden    | 主窗口隐藏，应用后台运行 |
| Visible   | 主窗口显示        |
| TopMost   | 主窗口显示并置顶     |
| Minimized | 主窗口最小化       |

快捷键触发逻辑：

```text
用户按下快捷键
│
├── 当前窗口隐藏
│   └── 显示窗口 → 置顶 → 聚焦
│
└── 当前窗口显示
    └── 隐藏窗口到托盘
```

## 5.2 全局快捷键模块

负责快捷键注册、冲突检测和配置保存。

功能要求：

* 用户可在设置页修改快捷键
* 支持 Ctrl / Alt / Shift / Win 组合键
* 支持字母键、数字键、功能键
* 快捷键保存后立即生效
* 快捷键冲突时提示用户重新设置
* 应用退出时释放快捷键

默认快捷键建议：

```text
Ctrl + Alt + Space
```

配置字段：

```json
{
  "hotkey": {
    "enabled": true,
    "modifiers": ["Ctrl", "Alt"],
    "key": "Space"
  }
}
```

## 5.3 待办事项模块

负责事项的增删改查、状态切换、日期筛选和排序。

### 5.3.1 新增事项

用户输入：

* 标题，必填
* 备注，选填
* 所属日期，默认今天
* 提醒时间，选填
* 是否启用每日继承提醒，默认启用

新增后：

* 事项出现在当天列表
* 状态为待处理
* 如果设置了提醒时间，则注册提醒任务

### 5.3.2 编辑事项

可编辑内容：

* 标题
* 备注
* 提醒时间
* 是否启用提醒
* 排序位置

不可随意编辑内容：

* 创建时间
* 继承来源
* 历史记录 ID

### 5.3.3 删除事项

删除分为两种：

1. 删除当天记录
2. 删除整个事项链

推荐默认行为：

* 删除当前日期的事项记录
* 如果该事项是继承事项，不影响历史日期记录
* 提供“删除全部历史记录”的二次确认选项

### 5.3.4 标记完成

用户点击完成后：

* 状态改为已完成
* 记录完成时间
* 取消当天未触发的提醒
* 不再继承到下一天

### 5.3.5 标记废弃

用户点击废弃后：

* 状态改为已废弃
* 记录废弃时间
* 取消当天未触发的提醒
* 不再继承到下一天

### 5.3.6 查询事项

支持按日期查询：

* 今日
* 昨日
* 指定日期
* 最近 7 天
* 最近 30 天

默认主界面展示：

```text
今日待办
```

## 5.4 日期继承模块

负责跨天时自动把未完成、未废弃事项继承到下一天。

触发时机：

1. 应用启动时
2. 日期变化时
3. 用户打开主窗口时
4. 每天 00:00 后首次检查时

继承判断逻辑：

```text
查找最近一个已存在待办记录的日期
│
├── 如果该日期早于今天
│   ├── 找出所有待处理事项
│   ├── 为今天创建新的待办记录
│   ├── 设置 inherited_from_record_id
│   ├── 保留标题、备注、提醒时间
│   └── 重置当天提醒触发状态
│
└── 如果今天已经完成继承
    └── 不重复继承
```

防重复继承规则：

* 每天只执行一次继承
* 同一个历史事项不能重复继承到同一天
* 数据库增加唯一约束防止重复记录

推荐唯一约束：

```text
task_id + record_date
```

## 5.5 事项提醒模块

负责扫描待办事项提醒时间，并在到点时弹出窗口。

提醒触发条件：

* 事项状态为待处理
* 已设置提醒时间
* 提醒时间小于等于当前时间
* 当天尚未触发过提醒

提醒弹窗内容：

* 事项标题
* 事项备注
* 提醒时间
* 操作按钮：

  * 知道了
  * 10 分钟后提醒
  * 30 分钟后提醒
  * 标记完成
  * 废弃

提醒触发后：

* 主窗口显示并置顶
* 对应事项高亮
* 写入提醒日志
* 更新 reminder_fired_at

稍后提醒逻辑：

```text
用户点击 10 分钟后提醒
│
└── 生成新的临时提醒时间 = 当前时间 + 10 分钟
```

## 5.6 定时弹出模块

定时弹出不同于事项提醒，它不绑定具体事项，而是用于按规则弹出主界面。

支持规则：

| 类型   | 示例                  |
| ---- | ------------------- |
| 单次弹出 | 2026-06-25 15:00 弹出 |
| 每日弹出 | 每天 09:00 弹出         |
| 每周弹出 | 每周一 09:00 弹出        |
| 间隔弹出 | 每 60 分钟弹出一次         |

弹出内容：

* 默认打开主窗口
* 显示今日待办
* 可显示一句提示语，例如“该查看今天的待办了”

定时弹出设置项：

* 是否启用
* 弹出类型
* 弹出时间
* 重复规则
* 提示文案
* 是否播放声音
* 是否显示系统通知

## 5.7 系统托盘模块

应用启动后常驻系统托盘。

托盘菜单包含：

```text
打开 / 隐藏
今日待办
新增事项
设置
退出
```

托盘行为：

* 单击：显示或隐藏主窗口
* 双击：显示主窗口并置顶
* 右键：打开菜单

退出应用时：

* 释放全局快捷键
* 停止后台定时器
* 保存配置
* 关闭数据库连接

## 8. 后台调度设计

应用后台需要同时处理三类任务：

1. 日期变化检查
2. 事项提醒检查
3. 定时弹出检查

推荐使用统一调度中心：

```text
SchedulerService
│
├── 每 30 秒检查一次事项提醒
├── 每 30 秒检查一次定时弹出
└── 每 60 秒检查一次日期变化
```

### 8.1 事项提醒检查

```text
当前时间 = now
查询今日待处理事项
筛选：
- reminder_enabled = true
- due_time <= 当前时间
- reminder_fired_at is null

对每条符合条件的事项：
- 弹出提醒
- 高亮事项
- 写入 reminder_fired_at
```

### 8.2 定时弹出检查

```text
查询启用状态的 popup_schedule
计算每条规则是否到达触发时间
如果到达：
- 显示主窗口
- 置顶
- 显示提示文案
- 更新 last_triggered_at
```

### 8.3 日期变化检查

```text
记录 last_checked_date
如果当前日期 > last_checked_date：
- 执行事项继承
- 更新 last_checked_date
```

## 10. 关键业务规则

### 10.1 事项状态规则

| 当前状态 | 可变更为 | 说明           |
| ---- | ---- | ------------ |
| 待处理  | 已完成  | 用户完成事项       |
| 待处理  | 已废弃  | 用户放弃事项       |
| 已完成  | 待处理  | 允许撤销完成       |
| 已废弃  | 待处理  | 允许恢复废弃       |
| 已完成  | 已废弃  | 不推荐直接切换，需先恢复 |
| 已废弃  | 已完成  | 不推荐直接切换，需先恢复 |

### 10.2 继承规则

待处理事项继承到下一天。

已完成事项不继承。

已废弃事项不继承。

被删除事项不继承。

已经继承过的事项不能重复继承。

### 10.3 提醒规则

事项只有在待处理状态下才会触发提醒。

已完成、已废弃、已删除事项不会触发提醒。

同一天同一事项默认只提醒一次。

用户选择稍后提醒时，会生成新的临时提醒时间。

### 10.4 窗口弹出规则

事项提醒优先级高于普通定时弹出。

如果同一时间既有事项提醒又有定时弹出，只显示事项提醒，同时主窗口打开。

如果多个事项同时到点，提醒弹窗显示事项列表。

## 11. 异常处理

### 11.1 快捷键注册失败

场景：

* 快捷键被其他应用占用
* 快捷键格式非法
* 系统拒绝注册

处理：

* 弹出提示
* 保留原快捷键
* 允许用户重新设置

### 11.2 数据库损坏

处理：

* 启动时检查数据库可用性
* 异常时提示用户
* 自动备份损坏数据库
* 创建新数据库
* 提供数据恢复入口

### 11.3 应用异常退出

处理：

* 下次启动时重新执行日期继承检查
* 根据数据库状态避免重复继承
* 未触发的提醒继续检查

### 11.4 电脑休眠后恢复

场景：

* 电脑休眠期间错过提醒时间

处理：

* 恢复后立即检查过期提醒
* 对当天已过期但未触发的提醒进行补提醒
* 弹窗提示“以下事项提醒时间已过”
